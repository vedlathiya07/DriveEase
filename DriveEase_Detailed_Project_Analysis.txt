## 1️⃣ Project Architecture Overview

DriveEase is a **MERN stack** application (MongoDB, Express.js, React.js, Node.js).
It facilitates car rentals, payments, status tracking, admin moderation, and map-based geolocation.

### Tech Stack Breakdown:
*   **Database:** MongoDB (using Mongoose as the ODM)
*   **Backend Interface:** Express.js (Node.js framework)
*   **Frontend Interface:** React.js (built with Vite)
*   **Payments:** Razorpay integration
*   **Image Storage:** Cloudinary
*   **Email Services:** Nodemailer / Resend
*   **Maps/Geolocation:** Leaflet / React-Leaflet

## 3️⃣ Database Models (`/models/`)

Models describe what data looks like when saved to MongoDB. You use **Mongoose Schemas**.

### 1. `User.js`
*   **Fields**: `firstName`, `lastName`, `email`, `password` (hashed), `role` (user or admin), `dateJoined`.
*   **Logic**: Before a user is saved, the password is encrypted using `bcrypt.js`.

### 2. `Car.js`
*   **Fields**: `brand`, `model`, `year`, `pricePerDay`, `availabilityStatus`, `images` (array of Cloudinary URLs), `location` (coordinates for the map interface), `owner` (reference to User).
*   **Logic**: Contains the blueprint for every vehicle available on the platform.

### 3. `Booking.js`
*   **Fields**: `user` (Reference to User), `car` (Reference to Car), `startDate`, `endDate`, `totalPrice`, `status` (pending, confirmed, cancelled), `paymentStatus`.
*   **Logic**: Tracks the relationship between a user and a rented car over a specific time period.

### 4. `PaymentSession.js`
*   **Fields**: `bookingId`, `razorpayOrderId`, `amount`, `status`, `createdAt`.
*   **Logic**: Stores intermediate logs of Razorpay transactions before validating the signature.

## 4️⃣ Backend Routing & Logic (`/routes/`)

Instead of writing all code in `app.js`, Express uses routers.

*   `routes/users.js`: 
    *   **POST** `/register`: Hashes passwords, stores users.
    *   **POST** `/login`: Checks passwords, generates a JWT (JSON Web Token) for authentication.
*   `routes/cars.js`:
    *   **GET** `/`: Fetches cars (with filters for search/availability).
    *   **POST** `/`: Allows uploading car details + images. Middleware (`multer`) processes the image and sends it to Cloudinary.
*   `routes/bookings.js`:
    *   **POST** `/`: Calculates days between `startDate` and `endDate`, multiplies by `pricePerDay`, creates the Booking, and triggers Razorpay order generation.
*   `routes/payment.js`:
    *   **POST** `/verify`: Webhook or frontend callback that receives Razorpay signatures. It hashes the internal secret against the signature. If they match, `paymentStatus` is set to "Completed".
*   `routes/admin.js`:
    *   Endpoints protected by an `isAdmin` middleware. Fetches system-wide stats, handles user bans or car deletions.

## 5️⃣ Frontend Architecture (`/client/`)

The React application is structured to be fast, modular, and component-based.

### 1. State Management (`client/src/context/`)
Since many components need to know "Is the user logged in?", we use the **React Context API**.
*   **`AuthContext.jsx`**: Wraps the whole app. It stores the `user` object and `token`. On component mount `useEffect()`, it checks local storage for a token. If found, it fetches the user details. Everything inside `AuthContext.Provider` can access this state.

### 2. Routing (`client/src/App.jsx`)
Uses `react-router-dom` to change the browser URL without refreshing the page.
*   **`<Routes>`**: Defines what page to load for what URL.
*   **Private Routes Wrapper**: Logic that says "If user is not logged in and tries to access `/booking`, redirect them to `/login`."

### 3. Pages (`client/src/pages/`)
Each file here represents a full screen in the app.
*   **`HomePage.jsx`**: Uses a hero section and maps over a shortened list of cars.
*   **`FleetPage.jsx`**: Retrieves the full `GET /api/cars/` array, manages search/filter states (`useState`), and renders grid items.
*   **`AdminHubPage.jsx`**: Makes authenticated requests to `api/admin` to render charts and data tables to oversee operations.

### 4. Components (`client/src/components/`)
Reusable UI pieces.
*   **`SiteNavbar.jsx`**: The header. Shows login/logout dynamically based on the AuthContext. Uses CSS classes for frosted glass UI.
*   **`CarCard.jsx`**: Takes a `car` object as a prop and renders a card with the image, name, and price.

### 5. Services Directory (`client/src/services/`)
This is where API calls are abstracted.
*   Instead of calling `fetch()` randomly in components, functions like `api.js` use `axios`.
*   An **Axios Interceptor** automatically attaches the user's JWT token to the `Authorization` header of *every* request, enforcing security sitewide.

## 6️⃣ Key Feature Workflows

### 🚗 Image Upload Flow
1. User selects an image in the React form.
2. Form is sent via `multipart/form-data` to `/api/cars`.
3. The Node server uses `multer-storage-cloudinary`.
4. Multer intercepts the request format, pipes the image directly into the Cloudinary servers.
5. Cloudinary returns a secure URL (`https://res.cloudinary.com/...`).
6. Node.js saves that URL to MongoDB under the `Car` images array.

### 💳 Payment & Booking Flow
1. React sends `startDate`, `endDate`, `carId` to Node.js.
2. Node creates a `Booking` marking it as "Payment Pending".
3. Node calls Razorpay API: "Create an order for ₹5000".
4. Razorpay returns an `order_id`. Node sends this to React.
5. React opens the Razorpay popup. User enters card details.
6. Razorpay processes and sends a token + signature back to React.
7. React forwards the signature to `/api/payment/verify`.
8. Node checks the math behind the signature cryptographic hash. If valid, booking goes to "Confirmed".

### 📍 Maps & Geolocation
1. Using Leaflet, coordinates (Latitude, Longitude) assigned to cars in MongoDB are passed to the frontend.
2. React renders `<MapContainer>` and places a `<Marker>` for each car.
3. If a car lacks coordinates, a fallback geocoding service converts their textual address into coordinates to place them accurately on the map.
